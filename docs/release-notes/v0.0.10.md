# Release v0.0.10

## New Features

### Shorten tool names exceeding 64 characters (#35)

MCP tool names (e.g. `mcp__plugin_chrome-devtools-mcp_chrome-devtools__get_network_request`) can exceed the Kiro backend's 64-character limit. Instead of rejecting with a hard error, kirocc now introduces bidirectional name mapping: `name[:50] + "_" + sha256(name)[:13]` = 64 chars.

- Names ≤ 64 chars pass through unchanged with zero overhead (lazy map initialization)
- Shortened names are sent to Kiro on the request path; original names are restored on the response path before returning to the Anthropic client
- Tool search history text also uses shortened names for consistency with tool schemas
