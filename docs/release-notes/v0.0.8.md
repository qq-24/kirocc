# Release v0.0.8

## New Features

### Use X-Claude-Code-Session-Id header as conversationId (#27)

Claude Code 2.1.86+ sends an `X-Claude-Code-Session-Id` header with every API request. The proxy now uses this header directly as the Kiro API `conversationId`, replacing the previous SHA1-based deterministic generation from model + system prompt + first user message.

The SHA1 approach was unreliable because system prompts change dynamically between turns, causing the conversation ID to drift within the same session. The header is now required on `/v1/messages` — requests without it receive a 400 error.

### Add session_id to request and response logs (#28)

The `session_id` (from `X-Claude-Code-Session-Id`) is now included in both request and response log lines, right after `trace_id`. This enables filtering logs by individual request (`trace_id`) or grouping by session (`session_id`).

## Bug Fixes

- Fix nullable anyOf/oneOf schemas being flagged with lossy warning (#26)

## Dependencies

- Update modernc.org/sqlite to v1.48.0 (#25)
- Bump google.golang.org/grpc from 1.79.2 to 1.79.3 (#24)
- Update modernc.org/sqlite to v1.47.0 (#23)

## Breaking Changes

- `X-Claude-Code-Session-Id` header is now required on `/v1/messages`. Clients that do not send this header will receive a 400 error.
