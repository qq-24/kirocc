# Release v0.0.5

## New Features

### Proxy-side Tool Search (regex / BM25) (#13)

Implement Anthropic's [Tool Search Tool](https://platform.claude.com/docs/en/agents-and-tools/tool-use/tool-search-tool) proxy-side since the Kiro backend does not support it natively. Supports both `tool_search_tool_regex_20251119` and `tool_search_tool_bm25_20251119` variants with `defer_loading` for on-demand tool discovery.

How it works:

1. Client sends a tool search tool definition + tools with `defer_loading: true`
2. Proxy partitions tools into active (sent to Kiro) and deferred (held for search)
3. Proxy injects a `ToolSearch` tool definition that Kiro can call
4. When the model calls `ToolSearch`, the proxy intercepts the tool_use, executes regex or BM25 search against deferred tools, emits `server_tool_use` + `tool_search_tool_result` SSE events to the client, and promotes discovered tools to active (up to 3 rounds)
5. When the model calls a regular tool or produces text, the response is forwarded normally

## Code Improvements

- Add block type constants (`BlockTypeToolUse`, `BlockTypeServerToolUse`, etc.) and `IsToolUse()`/`IsToolResult()` helpers to replace string literals across the codebase
- Conversation history `tool_reference` extraction for automatic tool promotion across turns
- Extract `writeStreamingOrJSONError` helper for reuse in tool search orchestrator
- Empty visible end_turn (thinking-only) retry support for the tool search path
