# Release v0.0.6

## New Features

### Effort Max Support via output_config (#17)

Claude Code now sends effort level via `output_config.effort` (max/high/medium/low). The previous implementation incorrectly used `thinking.reasoning_effort` which was not the correct field. This release fixes the parsing to use `output_config.effort` and maps effort `max` to budget 160,000 (same ~16% ratio as high=31999/200k applied to 1M context).

- Show effort level (max/high/medium/low/enabled) in thinking log instead of bare `true`
- Add debug-only client request body logging in `parseAndValidateRequest`, guarded behind `slog.LevelDebug`
- Add configurable `PORT` to Makefile run/debug targets (`make debug PORT=8001`)

## Code Improvements

- Add `CronCreate` workflow monitoring to release skill for automatic release workflow polling (#15)
